# AI-Driven Lunar Soil Composition Analysis Using Imagery and Large Language Models

**SUPARCO × Salim Habib University — Final Year Project Exhibition 2026**

An end-to-end AI pipeline that takes a single soil or lunar surface image and instantly delivers terrain classification, heavy metal composition estimates, and a Gemini-powered scientific report — all through a single-upload Streamlit web app.

---

## Overview

Traditional soil analysis requires physical sample collection followed by expensive ICP-MS/ICP-OES laboratory testing — a process that takes days and is infeasible for remote or hazardous environments. This project provides rapid preliminary screening from images alone using deep learning and large language models.

**Given one image, the system produces:**
- Terrain classification (Rocky Region / Crater / Big Rock / Artifact) with confidence scores
- Predicted concentrations of 6 heavy metals: Cd, Cu, Ni, Mn, Fe, Zn (in mg/kg)
- Safety status per element vs. WHO/FAO international soil guidelines
- A 7-section scientific report generated by Google Gemini 2.0 Flash
- PDF and Markdown download of the report

## Architecture

The system uses two independent ResNet-18 networks running in parallel on every uploaded image.

```
Input Image
    │
    ├──► Terrain CNN (ResNet-18 classifier)
    │        └─► 4-class softmax → Rocky Region / Crater / Big Rock / Artifact
    │
    └──► Composition CNN (ResNet-18 regressor)
             └─► 3-layer head → 6 normalised values → de-normalise → mg/kg
                                                              │
                                                    ┌─────────┴──────────┐
                                                    │  Gemini 2.0 Flash  │
                                                    │  (+ actual image)  │
                                                    └─────────┬──────────┘
                                                              │
                                                    Scientific Report (PDF)
```

### Composition Model

| Layer | Details |
|---|---|
| Backbone | ResNet-18 pre-trained on ImageNet (11M parameters) |
| Head | Linear(512→256) → BN → ReLU → Dropout(0.35) → Linear(256→128) → BN → ReLU → Dropout(0.20) → Linear(128→6) |
| Loss | Huber Loss |
| Optimiser | AdamW |
| Training | 80 epochs; backbone frozen for first 10, then unfrozen with 10× lower LR |
| Input size | 224 × 224 RGB |

### Terrain Model

| Layer | Details |
|---|---|
| Backbone | ResNet-18 (pre-trained, provided) |
| Head | Linear(512→4) + Softmax |
| Classes | Rocky Region, Crater, Big Rock, Artifact |

## Dataset

**Source:** SUPARCO Soil Academia Dataset

| Property | Value |
|---|---|
| Images | 180 JPEG (1280 × 960 px, RGB) |
| Ground truth | ICP-MS laboratory measurements |
| Elements | Cd, Cu, Ni, Mn, Fe, Zn |
| Unique compositions | 19 distinct chemical profiles |
| Images per profile | ~9 (same soil, different angles) |
| Labels file | `Soil_Analysis.xlsx` |

### Heavy Metal Safe Limits (WHO/FAO)

| Element | Safe Limit (mg/kg) |
|---|---|
| Cadmium (Cd) | 0.3 |
| Copper (Cu) | 36.0 |
| Nickel (Ni) | 35.0 |
| Manganese (Mn) | 400.0 |
| Iron (Fe) | 40,000.0 |
| Zinc (Zn) | 70.0 |

### Train / Validation Split

Split is done **by composition group** to prevent data leakage. All images of the same chemical profile go entirely into either training or validation — never both. This ensures the model is evaluated on compositions it has never seen.

```
19 groups → ~15 training groups + ~4 validation groups
```

### Label Normalisation

Labels are Z-score normalised before training (`mean` and `std` computed on training set only, saved in the model checkpoint). At inference, predictions are de-normalised back to mg/kg.

## Training Performance

Trained on Google Colab (T4 GPU), 80 epochs. Best checkpoint at epoch 60.

| Element | MAE (mg/kg) | RMSE (mg/kg) | R² |
|---|---|---|---|
| Cadmium (Cd) | 0.127 | 0.182 | -0.91 |
| Copper (Cu) | 0.566 | 0.690 | 0.16 |
| Nickel (Ni) | 0.501 | 0.723 | 0.33 |
| Manganese (Mn) | 7.136 | 8.678 | 0.54 |
| Iron (Fe) | 18.71 | 24.85 | 0.09 |
| Zinc (Zn) | 1.940 | 2.177 | -0.25 |

> **Note on R² scores:** The weak R² reflects the small dataset size (19 distinct compositions, ~4 validation groups) rather than a flaw in architecture. The Cd MAE of 0.127 mg/kg against a safe limit of 0.3 mg/kg demonstrates practically meaningful precision. Performance would improve significantly with a larger labelled dataset.

## How It Works

### Image Preprocessing

All uploaded images (JPG, PNG, 16-bit grayscale PCAM satellite imagery) are handled uniformly:

1. 16-bit images (`I;16` mode) are linearly normalised to 8-bit
2. Converted to RGB (handles grayscale, RGBA, palette modes)
3. Resized to 224 × 224
4. Converted to PyTorch tensor
5. Normalised with ImageNet statistics: `mean=[0.485, 0.456, 0.406]`, `std=[0.229, 0.224, 0.225]`

### Composition Prediction

The model outputs 6 normalised values. These are de-normalised using the training-set statistics stored in the checkpoint:

```
predicted_mg_kg = (model_output × label_std) + label_mean
predicted_mg_kg = clip(predicted_mg_kg, min=0)
```

### LLM Report

The `SoilLLMReporter` class sends a structured prompt to Gemini 2.0 Flash including:
- All 6 predicted concentrations with safe-limit comparisons
- Terrain class
- Model uncertainty (MAE values from training)
- The actual image (multimodal context)

Gemini returns a 7-section scientific report covering: executive summary, contamination assessment, soil health classification, environmental risk analysis, source analysis, remediation recommendations, and confidence/limitations.

If the Gemini API is unavailable, a rule-based fallback generates the report deterministically from threshold comparisons.
