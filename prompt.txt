------------------------------------------------------------
WHAT THIS PROJECT IS
------------------------------------------------------------
AI-Driven Soil Composition Analysis using Imagery and Large Language Models.
We analyse soil sample photographs and predict heavy metal concentrations
(Cadmium, Copper, Nickel, Manganese, Iron, Zinc) using a CNN regression model.
The system also classifies terrain types and generates scientific LLM reports.

------------------------------------------------------------
DATASET
------------------------------------------------------------
Location: Soil_data_Academia-20260601T122708Z-3-001/Soil_data_Academia/
- 180 JPEG images of soil samples (close-up photos of soil on white background)
- Soil_Analysis.xlsx: 180 rows, columns: Sample ID, Cd, Cu, Ni, Mn, Fe, Zn (mg/kg)
- 19 distinct composition profiles, ~9 images per profile (same sample, different magnifications)
- Naming: e.g. "1a_x", "1a_x1", "1a_x2", "1a_1_x" etc.

Terrain classes (4): Rocky Region, Crater, Big Rock, Artifact
- See classses_terrain.png at project root for visual examples
- Terrain model is ALREADY TRAINED: models/terrain_classifier.pth

STREAMLIT APP — 4 TABS
------------------------------------------------------------

Tab 1: Soil Composition Analysis
  - Upload any soil sample image
  - Predicts Cd, Cu, Ni, Mn, Fe, Zn values (mg/kg)
  - Shows status badges: Normal / Elevated / Critical vs WHO safe thresholds
  - Bar chart: predicted vs dataset mean vs safe threshold
  - Radar chart: element profile vs dataset mean
  - "Generate Scientific Report" button → Gemini LLM report
  - Fallback rule-based report if no API key

Tab 2: Terrain Classification
  - Upload landscape image → Rocky Region / Crater / Big Rock / Artifact
  - Shows confidence and all class probabilities
  - Uses models/terrain_classifier.pth (already trained)

Tab 3: Dataset Explorer
  - Box plots of all 6 element distributions
  - Correlation heatmap between elements
  - Sample browser: select sample → see its images + ground truth values

Tab 4: Model Performance
  - Shows R², MAE, RMSE per element (from training_metrics.json)
  - Heatmap of all metrics
  - Training + validation loss curve
  - If model not trained: shows instructions for Colab

------------------------------------------------------------
MODEL ARCHITECTURE
------------------------------------------------------------

CompositionModel (src/model.py):
  - ResNet-18 backbone (ImageNet pretrained)
  - Head: Linear(512→256) + BN + ReLU + Dropout(0.35) + Linear(256→128) + BN + ReLU + Dropout(0.2) + Linear(128→6)
  - Input: 224x224 RGB image
  - Output: 6 normalised values (de-normalised at inference time)
  - Loss: HuberLoss(delta=1.0) — robust to outliers
  - Training: backbone frozen for first 10 epochs, then unfrozen with lower LR

Training config (train.ipynb):
  - BATCH_SIZE = 16
  - EPOCHS = 80
  - LR_HEAD = 1e-3, LR_BACKBONE = 1e-4 (after unfreeze at epoch 10)
  - WEIGHT_DECAY = 1e-4
  - Val split: 20% (4 composition groups = ~44 images)
  - Train split: 80% (15 groups = ~131 images)
  - Split is by composition GROUP not image, to prevent data leakage

TerrainModel (already trained):
  - ResNet-18, 4 classes: Rocky Region, Crater, Big Rock, Artifact
  - Loaded from models/terrain_classifier.pth

------------------------------------------------------------
KEY TECHNICAL NOTES
------------------------------------------------------------

- ALWAYS use .venv Python: source .venv/bin/activate
  (system Python 3.14 does not have torch/pandas etc.)

- Dataset split: 19 composition groups → ~4 groups in val (~44 images), ~15 in train (~131)
  This prevents data leakage since same sample has multiple images

- Label normalisation: model outputs are normalised (mean/std from training set)
  De-normalisation is applied in inference.py using saved label_mean/label_std from checkpoint

- Safe thresholds in llm_reporter.py (WHO/FAO guidelines):
  Cd: 0.3, Cu: 36.0, Ni: 35.0, Mn: 400.0, Fe: 40000.0, Zn: 70.0 (all mg/kg)

- BatchNorm requires model.eval() for single-image inference (normal PyTorch behaviour)

- Gemini SDK: uses google.genai (new SDK), model: gemini-2.0-flash
